# Spring
开源的Java企业级应用开发框架，主要有两大核心思想IOC和AOP、
> DI（依赖注入）：组件所需要的依赖对象，不由自己new创建，而是由外部容器主动注入  
> IOC（控制反转）：把对象的创建、管理、生命周期控制权，从开发者代码反转交给Spring容器   
> AOP（面向切面编程）：在不修改原有业务代码的前提下，对方法进行横向增强，统一处理通用公共逻辑（纵向是业务主线，横向抽离通用切面）

| 对比维度    | Spring                                            | SpringBoot                                        | SpringCloud                                           |
|---------|:--------------------------------------------------|:--------------------------------------------------|:------------------------------------------------------|
| 核心定位    | Java企业级底层基础核心框架                                   | 基于Spring的快速开发脚手架/增强封装                             | 基于SpringBoot的微服务治理全家桶/规范套件                            |
| 核心能力    | 核心提供IOC容器、DI依赖注入、AOP面向切面编程三大基础能力，是所有Spring生态的底层基石 | 核心提供自动装配、内嵌Web容器、Starter场景化依赖、约定大于配置、简化Spring开发流程 | 核心提供服务注册发现、声明式远程调用、微服务网关、熔断降级限流、分布式配置、负载均衡等微服务全链路治理能力 |
| 解决的核心问题 | 解决传统 Java 开发的代码强耦合、对象管理繁琐、通用功能重复开发问题              | 解决 Spring 原生配置繁琐、项目搭建成本高、部署流程复杂的痛点                | 解决分布式 / 微服务架构下的服务通信、服务容错、流量管控、配置统一管理、服务治理等核心问题        |
| 配置复杂度   | 高，需要大量手动 XML / 注解配置，学习和维护成本高                      | 极低，基于自动装配，极少手动配置，开箱即用                             | 中等，仅需配置微服务治理相关规则，底层 Spring Boot 配置已大幅简化               |
| 依赖关系    | 无强制外部依赖，可独立运行                                     | 强制依赖 Spring 框架，无法脱离 Spring 独立运行                   | 强制依赖 Spring Boot，所有组件均基于 Spring Boot 构建，无法独立运行        |
| 适用场景    | 传统 Java Web 项目、基础组件开发、需要深度定制框架的场景                 | 单体架构 / 单个微服务节点，内嵌容器可直接运行                          | 分布式集群架构，多服务节点协同运行，需配套注册中心、网关等组件                       |
| 运行架构    | 传统单体架构、可独立部署                                      | 单体架构 / 单个微服务节点，内嵌容器可直接运行                          | 分布式集群架构，多服务节点协同运行，需配套注册中心、网关等组件                       |
| 核心特性    | 低侵入、高度解耦、一站式基础能力、生态兼容性强                           | 自动装配、内嵌容器、Starter 机制、约定大于配置、无代码生成、无 XML 配置        | 服务治理、流量管控、容错降级、分布式配置、链路追踪、开箱即用的微服务组件                  |
| 学习门槛    | 中高，需要理解 IOC/AOP 核心原理，掌握各类配置规则                     | 低，开箱即用，无需深入理解 Spring 底层即可快速开发                     | 中，需要理解微服务架构原理，掌握各类治理组件的使用和配置                          |
